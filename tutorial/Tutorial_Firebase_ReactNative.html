1. Introdução e Objetivos

Este tutorial demonstra como construir uma aplicação React Native completa utilizando o Firebase Firestore como banco de dados e o Firebase Authentication para autenticação de usuários.

O foco principal é a componentização e isolamento de funções, permitindo compreender exatamente como cada parte do app funciona.

A versão atual do projeto possui apenas três telas principais: Login, Cadastro e Home. Na Home o usuário visualiza seus dados, lista todos os usuários cadastrados no Firestore, edita registros, exclui registros e faz logout para voltar à tela de login.

O que você vai aprender

  • Criar e configurar um projeto no Firebase Console
  • Conectar um app React Native (Expo) ao Firebase
  • Implementar cadastro, leitura, edição e exclusão no Firestore
  • Isolar operações em serviços separados
  • Autenticar usuários com Firebase Auth
  • Navegar entre telas com React Navigation
  • Resolver o problema de scroll no web ajustando o layout raiz

2. Conceitos do Firebase

O que é o Firebase?

Firebase é uma plataforma da Google para desenvolvimento de aplicações. Oferece banco de dados, autenticação, armazenamento, hospedagem e muito mais, tudo gerenciado pela infraestrutura do Google Cloud.

O que é o Firestore?

O Cloud Firestore é um banco de dados NoSQL orientado a documentos. Os dados são organizados em coleções que contêm documentos, e cada documento pode ter campos de vários tipos.

ConceitoAnalogia SQLExemplo neste projeto
ColeçãoTabelausers
DocumentoLinha/RegistroDados de uma pessoa
CampoColunanome, email
ID do documentoChave primáriaGerado automaticamente

Exemplo de documento no Firestore

// Coleção: "users"
// Documento ID: "abc123xyz"

  "nome": "João Silva",
  "email": "joao@email.com",
  "telefone": "(11) 99999-9999",
  "idade": 28,
  "uid": "uid-do-firebase-auth"
}

4. Configurando o Firebase Console

1

Acessar o Firebase Console

Acesse https://console.firebase.google.com/ e faça login com sua conta Google.

2

Criar um novo projeto

  1. Clique em "Criar um projeto"
  2. Digite o nome do projeto (ex: sistema-firebase)
  3. Desative o Google Analytics (opcional para este tutorial)
  4. Clique em "Criar projeto" e aguarde
3

Adicionar um app Web

  1. Na tela inicial do projeto, clique no ícone "Web" (</>)
  2. Digite um apelido para o app (ex: app-react-native)
  3. Clique em "Registrar app"
  4. Copie o objeto firebaseConfig — você vai precisar dele!
// Exemplo do objeto que o Firebase fornece:
const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "meu-projeto.firebaseapp.com",
  projectId: "meu-projeto",
  storageBucket: "meu-projeto.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abc123"
};
4

Ativar o Firestore Database

  1. No menu lateral, clique em "Firestore Database"
  2. Clique em "Criar banco de dados"
  3. Selecione "Iniciar no modo de teste"
  4. Escolha a região "southamerica-east1 (São Paulo)"
  5. Clique em "Ativar"
⚠️ Modo de Teste O modo de teste permite leitura e escrita sem autenticação por 30 dias. Para produção, configure as regras de segurança adequadamente.
5

Ativar o Authentication

  1. No menu lateral, clique em "Authentication"
  2. Clique em "Começar"
  3. Na aba "Sign-in method", habilite "Email/senha"
  4. Clique em "Salvar"

7. Arquivo de Configuração — src/config/firebase.js

Este é o arquivo mais importante. Ele conecta o app ao seu projeto Firebase.

// src/config/firebase.js
import { initializeApp } from 'firebase/app';
import { getFirestore } from 'firebase/firestore';
import { getAuth, initializeAuth, getReactNativePersistence } from 'firebase/auth';
import { Platform } from 'react-native';
import AsyncStorage from '@react-native-async-storage/async-storage';

const firebaseConfig = { /* credenciais do projeto */ };

const app = initializeApp(firebaseConfig);

export const db = getFirestore(app);

export const auth =
  Platform.OS === 'web'
    ? getAuth(app)
    : initializeAuth(app, {
        persistence: getReactNativePersistence(AsyncStorage),
      });
⚠️ Importante: Substitua as credenciais Abra o arquivo src/config/firebase.js e substitua todos os valores "SUA_API_KEY", "SEU_PROJECT_ID" etc. pelos dados copiados do Firebase Console no Passo 4 da seção anterior.

O que cada função faz

FunçãoDescrição
initializeApp(config)Registra e inicializa o app Firebase com as credenciais
getFirestore(app)Retorna a instância do banco Firestore
getAuth(app)Usado no web para autenticação padrão
initializeAuth(app, options)Inicializa o serviço de autenticação no React Native
getReactNativePersistenceMantém o login salvo no dispositivo via AsyncStorage

8. Serviço de Autenticação — src/services/authService.js

Isola todas as funções de autenticação. As telas chamam essas funções sem precisar saber como o Firebase Authentication funciona internamente. 

// src/services/authService.js
import {
  createUserWithEmailAndPassword,
  signInWithEmailAndPassword,
  signOut,
} from 'firebase/auth';
import { auth } from '../config/firebase';

// CREATE USER — Cria novo usuário no Authentication
export const cadastrarUsuario = async (email, senha) => {
  const credencial = await createUserWithEmailAndPassword(auth, email, senha);
  return credencial.user;
};

// SIGN IN — Autentica o usuário com email e senha
export const fazerLogin = async (email, senha) => {
  const credencial = await signInWithEmailAndPassword(auth, email, senha);
  return credencial.user;
};

// SIGN OUT — Encerra a sessão do usuário atual
export const fazerLogout = async () => {
  await signOut(auth);
};

Descrição dos métodos

Método Firebase AuthO que faz
createUserWithEmailAndPasswordCria nova conta com email e senha. Retorna UserCredential.
signInWithEmailAndPasswordAutentica usuário existente. Lança erro se inválido.
signOutDesloga o usuário. auth.currentUser passa a ser null.
onAuthStateChangedListener que detecta mudanças no estado de autenticação.

11. Tela de Login — src/screens/LoginScreen.js

A tela de login valida email e senha e leva o usuário para a Home quando a autenticação é concluída.

Fluxo da tela

  1. Usuário digita email e senha
  2. Pressiona Entrar
  3. Chama fazerLogin(email, senha)
  4. Se sucesso, executa navigation.replace('Home')
  5. Se erro, exibe mensagem amigável
ℹ️ Validações usadas O projeto exige email válido e senha com pelo menos 6 caracteres, porque o Firebase Authentication rejeita campos inválidos.

12. Tela de Cadastro (CREATE) — src/screens/CadastroScreen.js

O cadastro cria a conta no Authentication e salva o perfil na coleção users.

Como funciona

  1. O usuário preenche nome, email, senha e campos opcionais
  2. Ao tocar em Enviar para o Firebase, o app cria a conta no Auth
  3. Depois salva o perfil em users/{uid}
  4. Ao finalizar com sucesso, navega para a Home
const usuario = await cadastrarUsuario(email, senha);
await salvarPerfilUsuario(usuario.uid, dadosPerfil);
navigation.replace('Home');

13. Tela Home — src/screens/HomeScreen.js

A Home reúne o que o app faz hoje: mostra os dados do usuário logado, lista todos os usuários cadastrados, permite editar/excluir e possui botão de sair.

Comportamento principal

  1. Busca o usuário autenticado com obterUsuarioAtual()
  2. Carrega o perfil atual com buscarPerfilUsuarioPorUid()
  3. Lista todos os perfis com listarPerfisUsuarios()
  4. Permite editar dados diretamente no card do usuário
  5. Permite excluir um registro com confirmação
  6. Executa fazerLogout() ao tocar em Sair da conta
⚠️ Scroll no web Como a Home é longa, o projeto recebeu um ajuste no App.js para liberar o scroll do body, do html e do #root no navegador.

16. Resumo dos Métodos Firestore

Operação Método Arquivo Tela
CREATE createUserWithEmailAndPassword + setDoc(users/{uid}) authService.js / userService.js CadastroScreen
READ getDoc() / getDocs() userService.js HomeScreen
UPDATE updateDoc() userService.js HomeScreen
DELETE deleteDoc() userService.js HomeScreen
AUTH signInWithEmailAndPassword authService.js LoginScreen

Comparativo: Firestore vs SQL

SQLFirestore
INSERT INTO users VALUES (...)setDoc(doc(db,'users',uid), dados)
SELECT * FROM usersgetDocs(collection(db,'users'))
UPDATE users SET nome=? WHERE uid=?updateDoc(doc(db,'users',uid), dados)
DELETE FROM users WHERE uid=?deleteDoc(doc(db,'users',uid))
