1. Introdução e Objetivos

Este tutorial demonstra como construir uma aplicação React Native completa utilizando o Firebase Firestore como banco de dados e o Firebase Authentication para autenticação de usuários.

O foco principal é a componentização e isolamento de funções, permitindo compreender exatamente como cada método do Firestore funciona.

O que você vai aprender

  • Criar e configurar um projeto no Firebase Console
  • Conectar um app React Native (Expo) ao Firebase
  • Implementar CRUD completo no Firestore
  • Isolar operações em serviços separados
  • Autenticar usuários com Firebase Auth
  • Navegar entre telas com React Navigation

2. Conceitos do Firebase

O que é o Firebase?

Firebase é uma plataforma da Google para desenvolvimento de aplicações. Oferece banco de dados, autenticação, armazenamento, hospedagem e muito mais, tudo gerenciado pela infraestrutura do Google Cloud.

O que é o Firestore?

O Cloud Firestore é um banco de dados NoSQL orientado a documentos. Os dados são organizados em coleções que contêm documentos, e cada documento pode ter campos de vários tipos.

ConceitoAnalogia SQLExemplo neste projeto
ColeçãoTabelapessoas
DocumentoLinha/RegistroDados de uma pessoa
CampoColunanome, email
ID do documentoChave primáriaGerado automaticamente

Exemplo de documento no Firestore

// Coleção: "pessoas"
// Documento ID: "abc123xyz"
{
  "nome": "João Silva",
  "email": "joao@email.com",
  "telefone": "(11) 99999-9999",
  "idade": 28,
  "criadoEm": Timestamp(2026-05-19)
}

7. Arquivo de Configuração — src/config/firebase.js

Este é o arquivo mais importante. Ele conecta o app ao seu projeto Firebase.

// src/config/firebase.js
import { initializeApp } from 'firebase/app';
import { getFirestore } from 'firebase/firestore';
import { initializeAuth, getReactNativePersistence } from 'firebase/auth';
import AsyncStorage from '@react-native-async-storage/async-storage';

// ⚠️ Substitua pelos dados do SEU projeto Firebase
const firebaseConfig = {
  apiKey: "SUA_API_KEY",
  authDomain: "SEU_AUTH_DOMAIN",
  projectId: "SEU_PROJECT_ID",
  storageBucket: "SEU_STORAGE_BUCKET",
  messagingSenderId: "SEU_MESSAGING_SENDER_ID",
  appId: "SEU_APP_ID",
};

// Inicializa o app Firebase com as configurações
const app = initializeApp(firebaseConfig);

// getFirestore() retorna a instância do banco de dados
export const db = getFirestore(app);

// initializeAuth com AsyncStorage mantém o login persistente
export const auth = initializeAuth(app, {
  persistence: getReactNativePersistence(AsyncStorage),
});
⚠️ Importante: Substitua as credenciais Abra o arquivo src/config/firebase.js e substitua todos os valores "SUA_API_KEY", "SEU_PROJECT_ID" etc. pelos dados copiados do Firebase Console no Passo 4 da seção anterior.

O que cada função faz

FunçãoDescrição
initializeApp(config)Registra e inicializa o app Firebase com as credenciais
getFirestore(app)Retorna a instância do banco Firestore
initializeAuth(app, options)Inicializa o serviço de autenticação
getReactNativePersistenceMantém o login salvo no dispositivo via AsyncStorage

8. Serviço de Autenticação — src/services/authService.js

Isola todas as funções de autenticação. As telas chamam essas funções sem precisar saber como o Firebase Authentication funciona internamente. 

// src/services/authService.js
import {
  createUserWithEmailAndPassword,
  signInWithEmailAndPassword,
  signOut,
} from 'firebase/auth';
import { auth } from '../config/firebase';

// CREATE USER — Cria novo usuário no Authentication
export const cadastrarUsuario = async (email, senha) => {
  const credencial = await createUserWithEmailAndPassword(auth, email, senha);
  return credencial.user;
};

// SIGN IN — Autentica o usuário com email e senha
export const fazerLogin = async (email, senha) => {
  const credencial = await signInWithEmailAndPassword(auth, email, senha);
  return credencial.user;
};

// SIGN OUT — Encerra a sessão do usuário atual
export const fazerLogout = async () => {
  await signOut(auth);
};

Descrição dos métodos

Método Firebase AuthO que faz
createUserWithEmailAndPasswordCria nova conta com email e senha. Retorna UserCredential.
signInWithEmailAndPasswordAutentica usuário existente. Lança erro se inválido.
signOutDesloga o usuário. auth.currentUser passa a ser null.
onAuthStateChangedListener que detecta mudanças no estado de autenticação.

9. Serviço CRUD — src/services/userService.js

Este é o coração do CRUD. Cada função representa exatamente uma operação no Firestore. As telas chamam essas funções sem conhecer a API do Firebase diretamente.

9.1 — CREATE: addDoc()

import { collection, addDoc, serverTimestamp } from 'firebase/firestore';

export const cadastrarPessoa = async (dados) => {
  // collection(db, 'pessoas') → referência à coleção "pessoas"
  const colecaoRef = collection(db, 'pessoas');
  
  // addDoc() adiciona um novo documento com ID automático
  const docRef = await addDoc(colecaoRef, {
    ...dados,
    criadoEm: serverTimestamp(), // Timestamp do servidor
  });
  
  return docRef.id; // Retorna o ID gerado
};
ℹ️ addDoc vs setDoc Use addDoc() quando quiser que o Firestore gere o ID automaticamente. Use setDoc(doc(db, 'col', 'meuId'), dados) quando quiser definir o ID manualmente.

9.2 — READ: getDocs()

import { collection, getDocs, query, orderBy } from 'firebase/firestore';

export const listarPessoas = async () => {
  const colecaoRef = collection(db, 'pessoas');
  
  // query() cria uma consulta com filtros (aqui: ordenar por nome)
  const consulta = query(colecaoRef, orderBy('nome'));
  
  // getDocs() busca todos os documentos que correspondem à consulta
  const snapshot = await getDocs(consulta);
  
  // .docs é um array de DocumentSnapshot
  // Mapeia cada doc para { id, ...campos }
  return snapshot.docs.map((doc) => ({
    id: doc.id,
    ...doc.data(),
  }));
};

9.3 — UPDATE: updateDoc()

import { doc, updateDoc, serverTimestamp } from 'firebase/firestore';

export const atualizarPessoa = async (id, dados) => {
  // doc(db, 'pessoas', id) → referência ao documento específico
  const docRef = doc(db, 'pessoas', id);
  
  // updateDoc() atualiza APENAS os campos enviados
  // Outros campos permanecem intactos no Firestore
  await updateDoc(docRef, {
    ...dados,
    atualizadoEm: serverTimestamp(),
  });
};

9.4 — DELETE: deleteDoc()

import { doc, deleteDoc } from 'firebase/firestore';

export const excluirPessoa = async (id) => {
  // doc(db, 'pessoas', id) → referência ao documento
  const docRef = doc(db, 'pessoas', id);
  
  // deleteDoc() remove o documento permanentemente
  await deleteDoc(docRef);
};
⚠️ deleteDoc() é irreversível! Sempre exiba um diálogo de confirmação (Alert.alert) antes de chamar deleteDoc(). O dado excluído não pode ser recuperado.

11. Tela de Login — src/screens/LoginScreen.js

Tela inicial do app. Autentica o usuário via Firebase Authentication.

Fluxo da tela

  1. Usuário digita email e senha
  2. Pressiona "Entrar"
  3. Chama fazerLogin(email, senha) do authService
  4. Se sucesso → navigation.replace('Listagem')
  5. Se erro → Alert.alert() com mensagem traduzida
// Trecho principal da LoginScreen:
const handleLogin = async () => {
  if (!email.trim() || !senha.trim()) {
    Alert.alert('Atenção', 'Preencha email e senha.');
    return;
  }

  setCarregando(true);
  try {
    await fazerLogin(email.trim(), senha);
    navigation.replace('Listagem'); // replace = remove Login da pilha
  } catch (erro) {
    // Trata erros específicos do Firebase Auth
    Alert.alert('Erro no Login', mensagemAmigavel);
  } finally {
    setCarregando(false);
  }
};
ℹ️ navigation.replace() vs navigation.navigate() replace('Listagem') substitui a tela atual na pilha, evitando que o usuário possa apertar "voltar" para retornar ao Login estando autenticado.

12. Tela de Cadastro (CREATE) — src/screens/CadastroScreen.js

Responsável por criar novos registros na coleção pessoas do Firestore usando addDoc().

Como funciona

  1. O usuário preenche nome, email, telefone e idade
  2. Ao pressionar "Cadastrar", chama cadastrarPessoa(dados)
  3. Internamente, addDoc() grava o documento no Firestore
  4. O ID gerado automaticamente é retornado e logado
  5. Exibe Alert de sucesso e navega para Listagem
// Trecho do handleCadastrar:
const dadosPessoa = {
  nome: nome.trim(),
  email: email.trim().toLowerCase(),
  telefone: telefone.trim(),
  idade: idade ? parseInt(idade, 10) : null,
};

// Chama addDoc() via userService
const idGerado = await cadastrarPessoa(dadosPessoa);
console.log('ID gerado:', idGerado);

await navigation.navigate('Listagem');

13. Tela de Listagem (READ + DELETE) — src/screens/ListagemScreen.js

Exibe todos os registros e permite excluir ou navegar para edição.

Detalhes técnicos importantes

useFocusEffect — Recarregar ao voltar

// useFocusEffect executa o callback SEMPRE que a tela
// recebe foco (inclusive ao voltar de outra tela)
useFocusEffect(
  useCallback(() => {
    setCarregando(true);
    carregarPessoas(); // chama getDocs() novamente
  }, [carregarPessoas])
);

Atualização otimista após exclusão

const handleExcluir = async (id) => {
  await excluirPessoa(id); // deleteDoc() no Firestore
  
  // Em vez de recarregar toda a lista, remove apenas o item local
  // Isso torna a UI mais responsiva (atualização otimista)
  setPessoas((prev) => prev.filter((p) => p.id !== id));
};

Pull-to-Refresh

// RefreshControl adiciona o gesto de puxar para atualizar
<FlatList
  refreshControl={
    <RefreshControl
      refreshing={atualizando}
      onRefresh={() => { setAtualizando(true); carregarPessoas(); }}
      colors={['#3F51B5']}
    />
  }
/>

14. Tela de Alteração (UPDATE) — src/screens/AlteracaoScreen.js

Recebe os dados da pessoa via parâmetros de navegação (route.params) e permite editar os campos.

Como os dados chegam nesta tela

// Na ListagemScreen, ao clicar em "Editar":
navigation.navigate('Alteracao', { pessoa }); // passa o objeto

// Na AlteracaoScreen, recebe via route.params:
const { pessoa } = route.params;

// Pré-preenche os campos com os dados existentes:
const [nome, setNome] = useState(pessoa.nome || '');

Como o update funciona

const dadosAtualizados = {
  nome: nome.trim(),
  email: email.trim().toLowerCase(),
  telefone: telefone.trim(),
  idade: idade ? parseInt(idade, 10) : null,
};

// updateDoc() atualiza APENAS os campos informados
// O campo "criadoEm" NÃO é alterado
await atualizarPessoa(pessoa.id, dadosAtualizados);
✅ Diferença entre updateDoc e setDoc
  • updateDoc(ref, dados) → Atualiza apenas os campos informados, preserva os demais
  • setDoc(ref, dados) → Substitui o documento inteiro (campos não informados são removidos)
  • setDoc(ref, dados, { merge: true }) → Funciona como updateDoc, mas cria o documento se não existir

17. Resumo dos Métodos Firestore

Operação Método Firestore Arquivo Tela
CREATE addDoc(colecaoRef, dados) userService.js CadastroScreen
READ getDocs(query(colecaoRef, orderBy(...))) userService.js ListagemScreen
UPDATE updateDoc(docRef, dados) userService.js AlteracaoScreen
DELETE deleteDoc(docRef) userService.js ListagemScreen
AUTH signInWithEmailAndPassword authService.js LoginScreen

Comparativo: Firestore vs SQL

SQLFirestore
INSERT INTO pessoas VALUES (...)addDoc(collection(db,'pessoas'), dados)
SELECT * FROM pessoas ORDER BY nomegetDocs(query(ref, orderBy('nome')))
UPDATE pessoas SET nome=? WHERE id=?updateDoc(doc(db,'pessoas',id), dados)
DELETE FROM pessoas WHERE id=?deleteDoc(doc(db,'pessoas',id))
